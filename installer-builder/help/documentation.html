Installer Builder Help Guide

This guide explains how to use the Installer Builder to create professional, standalone Windows installers for your applications.

NOTE: Since this executable (installer_builder.exe) is unsigned, you will see a warning from Windows SmartScreen warning you about it. You can bypass this by clicking "More info" then "Run anyway".

NOTE: installer_builder.exe will be flagged as a virus by Google and other engines. This is because the installers it generates interact with a system folder (Program Files) and modify the Windows Registry (to make uninstallation clean). We assure you this is a false positive and that installer_builder.exe is completely safe to run.

1. Getting Started: The Setup Process

Before using the Builder, your system must be prepared. The install_neccesary_programs.cmd script automates this entire process.

What install_neccesary_programs.cmd does:
  • Checks for Python 3.12 and installs it silently if missing.
  • Automatically adds Python to your system PATH so commands work instantly.
  • Downloads and installs PyInstaller, the engine used to package your apps.
  • Installs pypiwin32 to allow the creation of Start Menu and Desktop shortcuts.
  • NOTE: install_neccesary_programs.cmd does not require admin rights. It can be run by any user. However, it does require an internet connection.

2. Using the Builder Executable

The installer_builder.exe is a high-performance, machine-code application. It provides a graphical interface to package your folders.

  • Payload Folder: The folder containing your game or app files.
  • Output Path: Where you want the final setup.exe to be saved.
  • Shortcut Name: The name users will see in their Start Menu.

3. How the Generated Installer Works

When you share your generated installer with others, it performs several tasks:

  • File Extraction: It installs your app and its files to Program Files (for Admins) or AppData (for standard users).
  • Registry Integration: It creates a key in HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Uninstall. This ensures your app appears in the "Add or Remove Programs" list.
  • Uninstaller Creation: It bundles a dedicated uninstaller.exe that can safely remove the app, its shortcuts, and its registry entries later.
Note on Security: The builder includes a pre-build "Mock Scan." It is recommended to verify your files with an antivirus before packaging to ensure a safe experience for your users.

4. Troubleshooting

If the "Build" button fails, ensure you have run install_neccesary_programs.cmd at least once with an active internet connection to download the required build engines.
